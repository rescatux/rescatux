Check if there is a bios_grub partition on a GPT disk - Rescapp - Documentation

Do you actually need a bios_grub partition ? It depends. A bios_grub partition is usually needed if you use grub-pc on a GPT disk. If your computer has both a GPT disk and EFI boot support you might be better on trying to install grub-efi package. If you need to use grub-pc package on a GPT disk after all this option let's you find if your targeted hard disk has the bios_grub partition or not. If you think that you need to create a bios_grub partition please ask to your distribution usual support places. An option to help you to create a bios_grub partition is not on the short term Rescatux TODO list.
  • Step 1

    Rescapp searchs for hard disks on your computer.
  • Step 2

    If such hard disks have been found Rescapp presents you the hard disks. At the same time you are prompted:

    Which hard disk?

    You are supposed to choose the hard disk which has a GPT partition which you want its bios_grub partition to be checked.
  • Step 3

    Rescapp with all gathered information is going to check if there is a bios_grub partition inside your GPT hard disk.Please wait. Currently (Rescatux 0.40b1) this step is time consuming but no Please wait message has been implemented.
  • Step 4

    You are informed about the bios_grub checking process.

    Message is self explanatory. You either get:

    bios_grub partition was found! :)

    or

    bios_grub partition was not found. :(
  • Step 5

    A log file called gpt-check-bios-grub_log.txt can be found at Desktop logs folder.
