Recompute CHS on an hybrid MBR inside a GPT partition - Rescapp - Documentation

This option can sometimes help if a disk utility, OS, or BIOS doesn't like the CHS values used by the partitions in the protective or hybrid MBR. In particular, the GPT specification requires a CHS value of 0xFFFFFF for over-8GiB partitions, but this value is technically illegal by the usual standards. Some BIOSes hang if they encounter this value. This option will recompute a more normal CHS value -- 0xFEFFFF for over-8GiB partitions, enabling these BIOSes to boot.
  • Step 1

    Rescapp searchs for hard disks on your computer.
  • Step 2

    If such hard disks have been found Rescapp presents you the hard disks. At the same time you are prompted:

    Which hard disk?

    You are supposed to choose the hard disk which has a GPT partition which you want its hybrid MBR CHS to be recomputed.
  • Step 3

    Rescapp with all gathered information is going to recompute the CHS for the hybrid MBR inside your GPT hard disk.Please wait. Currently (Rescatux 0.40b1) this step is time consuming but no Please wait message has been implemented.
  • Step 4

    You are informed about the hybrid mbr CHS recompute process.

    Message is self explanatory. You either get:

    Success! Hybrid MBR CHS was recomputed OK! :)

    or

    Failure! Hybrid MBR CHS was not recomputed. Something went wrong! :(
  • Step 5

    A log file called gpt-recompute-hybrid-mbr-chs_log.txt can be found at Desktop logs folder.
