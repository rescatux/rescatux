Change Gnu/Linux password - Rescapp - Documentation
  • Step 1

    Rescapp searchs for partitions that have on them the following file: /etc/issue based on information found in /proc/partitions file.
  • Step 2

    If such partitions have been found Rescapp presents you partitions that have on them the following file: /etc/issue. At the same time you are prompted:

    Which partition is your main GNU/Linux?

    You are supposed to choose your main GNU/Linux distribution (probably you only will see one) and press the OK button.
  • Step 3

    You are prompted:

    Which user?

    Select the desired user from the list.
  • Step 4

    You are prompted:

    Enter password (user)

    Use a temporal password please. You will change it later on your system.
    • Step 5

    Rescapp tries to changes the password for the selected user.
  • Step 6

    You are informed about the password change process.

    Message is self explanatory. You either get:

    Password was changed OK! :). We recommend to change it once you're back in your system to use your system default encryption

    or

    Failure! Password was not changed. Something went wrong! :(
  • Step 9

    A log file called chpasswd.log.txt can be found at Desktop logs folder.
